# Emocre Site — Marketing & Creature Compendium
Static site for Emocre, built with **Eleventy 3 + SCSS (Dart Sass)**. Served at the domain in `CNAME` (`emocre.com`). It showcases the creature roster (data + art) and the press kit, all sourced from Notion and the `emocre-art` repo — nothing about creatures is authored here by hand.

## Critical Rules

1. **NEVER edit generated/imported content** — re-run the step that produces it:
   - `data/` — imported from Notion by `make import`. **gitignored.**
   - `_data/creatures.json` — produced by `make transpile` (`tools/transpile.ts`). Edit the transpiler, not the JSON.
   - `assets/creatures/*` — creature images rendered by `make transpile` (CRT filter; animated emotions become GIFs from Aseprite strips).
   - `_site/` — Eleventy build output. **gitignored.**
2. **NEVER edit `vendor/`** — `vendor/emocre-art` and `vendor/emocre-tools` are cloned by `make install` and **gitignored**. Fix shared code in the `emocre-tools` repo; fix art in `emocre-art`.
3. **Hand-authored sources are fair game:** `_data/site.json` (site config), `_includes/` (templates), `assets/css` (SCSS), `assets/js`, `index.html`, `play.html`, `press-kit.md`.

## Project Structure

```
index.html  play.html        # pages
press-kit.md                  # press kit (also synced from Notion on import)
_data/
  site.json                   # site config (title, url, lang) — SOURCE
  creatures.json              # ⚠️ GENERATED by make transpile
_includes/                    # Eleventy templates/partials
assets/
  creatures/                  # ⚠️ GENERATED creature images (png/gif)
  css/                        # compiled from SCSS
  js/
data/                         # ⚠️ Notion import (creature/move/type) — gitignored
tools/                        # transpile.ts, paths.ts (emocre-specific paths)
vendor/
  emocre-art/                 # sparse-checkout of `art/` only (game-ready creature PNGs)
  emocre-tools/               # @emocre/tools workspace
_site/                        # ⚠️ Eleventy build output — gitignored
eleventy.config.mjs  _config.yml  CNAME
```

## Data Workflow

```
Notion ──make import──▶ data/ (YAML) ─┐
emocre-art (vendor) art/ ─────────────┴─make transpile──▶ _data/creatures.json + assets/creatures/*
                                                          └─make build / start──▶ _site/ (Eleventy)
```

`make transpile` (`tools/transpile.ts`) reads the type chart from `data/`, pulls creatures flagged `site` from the vendored `art/` via `@emocre/tools`'s `CreaturesFileRepository`, computes type weaknesses, applies the CRT filter to front art (GIF for animated `.ase`), and writes `_data/creatures.json`.

> Because transpile reads `vendor/emocre-art/art/`, the site depends on `emocre-art` exposing an `art/` directory (pulled via `git sparse-checkout set art`). If that repo's layout changes, `make install` and the transpiler must be updated to match.

## Commands

```bash
make install      # Clone/update vendor (emocre-tools + sparse art) + yarn install
make import       # Pull creatures/types + press-kit from Notion into data/
make transpile    # Build _data/creatures.json + assets/creatures from data/ + vendored art
make build        # Eleventy production build → _site/
make start        # install + import + transpile + Eleventy dev server (watch)
yarn build / yarn start   # Eleventy only (no data refresh)
```

## Code Conventions

- TypeScript tooling runs via `ts-node` (`transpileOnly`, `strict: false`); `camelCase`/`PascalCase`.
- Emocre-specific paths live in `tools/paths.ts` — keep path knowledge there, not scattered through the transpiler.
- Eleventy formats: `scss`, `html`, `md` (see `package.json` scripts).
- UI copy is **pt-BR** (see `_data/site.json`); creature names/descriptions come from Notion in PT.

## Environment

`.env`: `NOTION_API_KEY=<key>` (consumed by `make import` via `@emocre/tools`).
